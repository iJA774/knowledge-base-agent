# 🧠 OCAM: Omni-Cognitive Agent Matrix 

[![Python 3.10+](https://img.shields.io/badge/python-3.10+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**OCAM** (Omni-Cognitive Agent Matrix) is an enterprise-grade, fully automated personal/team knowledge base driven by Swarm Intelligence. 

Traditional RAG (Retrieval-Augmented Generation) systems suffer from shallow semantic matching, context loss, and severe model hallucinations when faced with complex, cross-domain reasoning tasks. OCAM solves this by abandoning single-shot generation in favor of a **Heavy-Duty Multi-Agent Debate Protocol**, **Graph-Enhanced Retrieval (GraphRAG)**, and **Tree of Thoughts (ToT)** reasoning.

By trading high compute (Token usage) for absolute logical rigor, OCAM achieves near zero-hallucination knowledge synthesis, acting as a tireless, deep-dive research assistant.

---

## 🔥 Key Features

### 1. Deep Knowledge Ingestion Pipeline
We don't just chunk text; we reconstruct it.
* **Graph Entity Extraction:** Scans documents paragraph by paragraph to extract entities, claims, and logical relationships, mapping them into a multidimensional graph space.
* **Recursive Hierarchical Summarization:** Utilizes a Map-Reduce architecture to generate bottom-up summaries, ensuring no micro-details or macro-contexts are lost during ingestion.

### 2. Swarm Intelligence QA Network
Answers are not generated; they are debated and synthesized.
* **Task Decomposition:** A `Planner Agent` breaks down complex user queries into orthogonal sub-tasks.
* **Parallel Hybrid Retrieval:** `Researcher Agents` execute parallel queries across Vector DBs (Chroma/Milvus) and metadata structures.
* **Red-Teaming & Fact-Checking:** A dedicated `Critic Agent` aggressively challenges the draft generated by the `Synthesizer Agent`. Every factual claim is cross-referenced against the source context. If a claim lacks citation, it is rejected and rewritten.

### 3. Autonomous Reflection & Evolution
* **Nightly Memory Consolidation:** During idle hours, a background `Reflection Agent` (LLM-as-a-Judge) reviews daily interactions, scoring them for accuracy and fluency, and fuses high-frequency knowledge into "Super Memory Chunks" for optimized future retrieval.

## 🏗️ System Architecture

```text
User Query ──> [Router Agent] 
                   │
                   ├──> Intent: Ingest ──> [Cleaner] ─> [Graph Extractor] ─> [Vector Store & Metadata]
                   │
                   └──> Intent: Query ──> [Planner] ─> 1..N [Researchers]
                                                            │
                                                     [Hybrid Retrieval]
                                                            │
                            [Final Output] <── [Critic] <── [Synthesizer (ToT)]
```

## 🚀 Getting Started

### Prerequisites
* Python 3.10+
* A valid API key for high-tier LLMs (OpenAI GPT-4o, Claude 3.5 Sonnet, etc., are highly recommended due to the complex reasoning required).

### Installation

1. Clone the repository:
   ```bash
   git clone [https://github.com/yourusername/OCAM.git](https://github.com/yourusername/OCAM.git)
   cd OCAM
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Set up your environment variables:
   ```bash
   cp .env.example .env
   # Edit .env and add your OPENAI_API_KEY
   ```

### Usage

Run the autonomous matrix terminal:

```bash
python main.py
```
* **To ingest knowledge:** Simply paste articles, reports, or notes. The Router Agent will automatically detect the ingest intent and trigger the Deep Ingestion Pipeline.
* **To query:** Ask complex, multi-layered questions. You will see the live logs of Agents planning, researching, and debating before delivering the final synthesized answer.

---

## ⚠️ Warning: High Token Consumption

**Please Note:** OCAM is designed for extreme accuracy and depth, not cost-efficiency. 

Because every single query triggers a cascade of Agent interactions (Planning → Parallel Retrieval → Drafting → Red-Teaming → Revision), a single complex question can consume **20k - 50k tokens**. Batch ingestion of reports using the Recursive Summarization protocol will consume multiples of the original document length. 

Ensure your API tier and budget are configured to handle heavy, sustained loads before running OCAM in a production environment.
